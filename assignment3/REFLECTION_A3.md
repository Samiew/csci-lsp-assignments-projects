# Assignment 3 Reflection – ETL Pipeline with AI

## What I Built
For this assignment, I built an ETL (Extract, Transform, Load) pipeline in Java. The program reads product data from a CSV file, processes and transforms the data based on specific rules, and writes the cleaned data to a new file.

## Key Features
- Reads input data from a CSV file
- Skips invalid or malformed rows
- Transforms product names to uppercase
- Applies a 10% discount to electronics
- Categorizes products into price ranges (Low, Medium, High, Premium)
- Outputs clean, structured data into a new CSV file

## Challenges
One challenge I faced was ensuring that invalid data did not crash the program. I had to carefully handle exceptions and validate input before processing it. Another challenge was correctly implementing rounding and price calculations using BigDecimal.

## Use of AI
I used AI as a learning tool to help understand how to structure the ETL pipeline and improve my Java syntax. AI assisted in refining my approach, but I made sure to test, understand, and verify all parts of the code before using them.

## What I Learned
Through this assignment, I gained a better understanding of file handling in Java, data transformation techniques, and how to build structured and reliable programs. I also learned how to use AI responsibly as a support tool rather than relying on it blindly.
